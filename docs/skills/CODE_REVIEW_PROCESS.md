# Code Review Process Skill

## 目的
- レビュー観点を統一し、重要な不具合を早期に発見する

## 入力テンプレート
```
対象: [PR/コミット/ファイル]
スコープ: [全体/特定モジュール]
重点: [安全性/性能/可読性]
```

## チェックリスト
- [ ] 仕様通りの挙動
- [ ] エッジケース対応
- [ ] エラー処理
- [ ] パフォーマンス
- [ ] セキュリティ
- [ ] テストの有無
- [ ] ドキュメント更新

## 出力テンプレート
```
結論: [Approve/Request Changes/Comment]
重要指摘:
- [高優先度の指摘]
軽微な指摘:
- [改善案]
テスト確認: [結果/未実施]
```
