# Browser Automation Debug Skill

## 目的
- ブラウザ自動化の失敗を再現・切り分け・修正する

## 前提
- 対象ブラウザ: [Chrome/Chromium]
- 実行環境: [ローカル/Dev Container/CI]
- 使用ツール: [Chrome DevTools/Playwright/Puppeteer]

## 入力テンプレート
```
対象: [対象フロー/URL]
再現条件: [入力/状態/アカウント]
期待動作: [正しい動作]
実際の動作: [失敗内容]
頻度: [常に/時々]
ログ: [取得済みログのパス]
```

## 手順チェックリスト
- [ ] 再現性の確認（最小入力で再現できるか）
- [ ] 画面キャプチャ取得（動画/スクショ）
- [ ] コンソールエラーの記録
- [ ] ネットワークログ取得（HAR）
- [ ] UI要素のセレクタ再確認
- [ ] 待機条件の確認（明示的待機）
- [ ] 依存データの確認（テストデータ/セッション）

## 成果物
- 原因候補と根拠
- 再現手順
- 修正案（最小差分）

## 出力テンプレート
```
結論: [最有力の原因]
根拠: [ログ/再現条件]
再現手順: [最小再現]
修正案: [対応策]
```
